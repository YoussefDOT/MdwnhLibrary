## لماذا نحتاج كل هذا؟

الموقع صفحة ثابتة على GitHub Pages. الصفحة الثابتة لا تستطيع إرسال تنبيه وهي مغلقة — لأنها ببساطة لا تعمل عندما تُغلق.

الحل يتكوّن من ثلاث قطع:

| القطعة | أين تعيش | وظيفتها |
|---|---|---|
| **Service Worker** (`sw.js`) | داخل جهاز العضو | يبقى نائمًا، ويستيقظ عند وصول تنبيه ويعرضه |
| **الاشتراك** (subscription) | Firebase | عنوان بريدي فريد لكل جهاز، نرسل إليه التنبيه |
| **المُرسِل** (`tools/send-reminders.mjs`) | GitHub Actions | يعمل كل ٥ دقائق، يقرأ المهام ويرسل التنبيهات |

القطعتان الأولى والثانية جاهزتان بالفعل. أنت فقط تحتاج تشغيل الثالثة.

**التكلفة: صفر.** GitHub Actions مجاني بلا حدود للمستودعات العامة (public).

## الخطوة ١ — أنشئ مفاتيح VAPID

مفاتيح VAPID هي بطاقة هوية موقعك عند خدمات التنبيه (Google، Apple، Mozilla). مفتاحان: **عام** (يوضع في الموقع) و**خاص** (سرّ، لا يُنشر أبدًا).

افتح Terminal على جهازك:

```bash
cd "/Volumes/Sigma Drive/Vibe Coding/MdwnhLibrary/tools"
npm install
npm run keys
```

سيطبع شيئًا مثل:

```
=======================================
Public Key:
BEl62iUYgUivxIkv69yViEuiBIa-Ib9-SkFYs...

Private Key:
UUxI4O8-FbRouAevSmBQ6o18hgE4nSG3qwvJT...
=======================================
```

⚠️ **لا تغلق النافذة.** ستحتاج القيمتين الآن.

> **تحذير أمني:** المفتاح الخاص (Private Key) سرّ. لا تضعه في `library.js` ولا في أي ملف داخل المستودع، ولا ترسله في رسالة. من يملكه يستطيع إرسال تنبيهات باسم موقعك. مكانه الوحيد هو GitHub Secrets في الخطوة ٣.

## الخطوة ٣ — احفظ المفاتيح في GitHub Secrets

| Name | Secret |
|---|---|
| `VAPID_PUBLIC` | المفتاح العام |

## الخطوة ٤ — فعّل GitHub Actions

1. في المستودع، اضغط تبويب **Actions**
2. إن ظهرت رسالة تطلب تفعيل الـ workflows، اضغط **I understand my workflows, go ahead and enable them**
3. من القائمة اليمنى اختر **Task reminders**
4. اضغط **Run workflow** ← **Run workflow** لتجربتها يدويًا

انتظر نحو دقيقة ثم اضغط على التشغيل لرؤية السجل. المتوقع:

```
No tasks. Nothing to do.
```
أو
```
Done. 0 notification(s) delivered.
```

هذا نجاح. (صفر لأنه لا يوجد أحد مشترك بعد.)

## متى تصل التنبيهات؟

| الحالة | التوقيت |
|---|---|
| مهمة جديدة — **مع Vercel** (الخطوة ٧) | خلال ثانية تقريبًا |
| مهمة جديدة — بالجدولة وحدها | خلال ١٥–٤٠ دقيقة |
| بقي يوم واحد على الموعد | مرة واحدة عند دخول آخر ٢٤ ساعة |
| أتممت المهمة | لا شيء — من أتمّ المهمة لا يُذكَّر بها |

الـ cron مضبوط على `7,22,37,52 * * * *` — أي كل ١٥ دقيقة، وبإزاحة مقصودة عن رأس الساعة.

### ⚠️ لا تُنقصها إلى `*/5`

جرّبناها. GitHub يقول في توثيقه إن أقصر مدة هي ٥ دقائق، لكنه **لا يلتزم بذلك عمليًا**. المهام المجدولة كل أقل من ١٥ دقيقة على المستودعات قليلة النشاط تُهمَل ويُسقطها الجدول بصمت — بقيت ٤٥ دقيقة بلا تشغيل واحد. كذلك الدقائق `:00` و`:15` و`:30` و`:45` هي الأكثر ازدحامًا، ولهذا اخترنا `7,22,37,52`.

التشغيل اليدوي (`workflow_dispatch`) يتجاوز الجدول تمامًا، ولهذا ينجح دائمًا. نجاحه لا يعني أن الجدولة تعمل.

لتسريع الاختبار: **Actions** ← **Task reminders** ← **Run workflow**.

## الخطوة ٧ — التنبيه الفوري (Vercel)

الجدولة وحدها تعني انتظار ١٥–٤٠ دقيقة. هذه الخطوة تجعل تنبيه «مهمة جديدة» يصل **خلال ثانية تقريبًا**.

### الفكرة

عندما يضيف نواف مهمة، متصفحه يُرسل رقم المهمة فقط إلى دالة صغيرة على Vercel. الدالة تقرأ المهمة من Firebase بنفسها وترسل التنبيه فورًا.

الجدولة تبقى شغّالة كشبكة أمان: إن فشل الاتصال الفوري لأي سبب، تلتقطها الجدولة لاحقًا. **المهمة قد تتأخر، لكنها لا تضيع أبدًا.**

### التنفيذ

**١. أنشئ حسابًا مجانيًا على <https://vercel.com>** (سجّل الدخول بحساب GitHub — أسهل).

**٢. ثبّت أداة Vercel وادخل:**

```bash
npm install -g vercel
vercel login
```

**٣. ارفع الدالة:**

```bash
cd "/Volumes/Sigma Drive/Vibe Coding/MdwnhLibrary/notify-api"
vercel --prod
```

سيسألك أسئلة — اضغط Enter لقبول الافتراضي في كلها، وعند `Link to existing project?` اختر **No**.

في النهاية سيطبع رابطًا مثل:

```
https://notify-api-xxxx.vercel.app
```

احتفظ به.

**٤. أضف المفاتيح إلى Vercel:**

```bash
vercel env add VAPID_PUBLIC production
vercel env add VAPID_PRIVATE production
vercel env add VAPID_SUBJECT production
```

سيطلب منك القيمة بعد كل أمر — الصق نفس القيم الثلاث التي وضعتها في GitHub Secrets.

ثم أعد الرفع حتى تُطبَّق المفاتيح:

```bash
vercel --prod
```

**٥. ضع الرابط في الموقع.** افتح `library.js` وابحث عن:

```js
var NOTIFY_URL = '';
```

ضع رابطك متبوعًا بـ `/api/notify`:

```js
var NOTIFY_URL = 'https://notify-api-xxxx.vercel.app/api/notify';
```

**٦. ارفع التعديل:**

```bash
cd "/Volumes/Sigma Drive/Vibe Coding/MdwnhLibrary"
git add -A && git commit -m "feat: instant task notifications" && git push
```

### التحقق

أضف مهمة من حساب نواف وكلّف بها نفسك. يجب أن يصلك التنبيه خلال ثانية أو ثانيتين.

إن لم يصل، افتح **Console** في المتصفح وابحث عن خطأ من `notify`. أو جرّب الدالة مباشرة:

```bash
curl -X POST https://notify-api-xxxx.vercel.app/api/notify \
  -H "Content-Type: application/json" \
  -d '{"taskId":"ضع-رقم-المهمة-هنا"}'
```

الرد المتوقع: `{"sent":1,"assignees":1}` أو `{"skipped":"already notified"}`.

### هل الرابط مكشوف؟

نعم، وهذا مقصود ولا يشكّل خطرًا:

- الدالة تقبل **رقم مهمة فقط** — لا عنوانًا ولا قائمة مستقبلين. كل شيء يُقرأ من Firebase.
- الدالة **لا تكرّر** — بعد أول إرسال تُعلّم المهمة ولا ترسل ثانية.

أقصى ما يستطيعه شخص غريب هو إعادة إطلاق تنبيه كان سيُرسَل أصلًا، مرة واحدة. لهذا لا تحتاج كلمة سر في الموقع.

### ⚠️ إن غيّرت صيغة الرسالة

نص التنبيه مكتوب في مكانين ويجب أن يتطابقا:

- `notify-api/api/notify.js` (الفوري)
- `tools/send-reminders.mjs` (المجدول)

غيّرت في أحدهما؟ غيّر في الآخر.

---

### إن ظلّت الجدولة لا تعمل

جدولة GitHub ليست مضمونة إطلاقًا — لا يوجد اتفاق خدمة عليها في الخطة المجانية. إن لاحظت أنها تتوقف كثيرًا، البديل الموثوق هو **Cloudflare Workers** مع Cron Triggers: مجاني، ويعمل بدقة كل دقيقة، ويستطيع إرسال Web Push بنفس مفاتيح VAPID. يحتاج نقل `send-reminders.mjs` إلى Worker. أخبرني وقتها وأنقله لك.

## ⚠️ الآيفون — مهم جدًا

سفاري على الآيفون **لا يدعم التنبيهات** إلا إذا كان الموقع **مثبَّتًا كتطبيق** على الشاشة الرئيسية. لا توجد طريقة تجاوز — هذا قيد من Apple.

أخبر أعضاء الفريق على الآيفون بالخطوات:

1. افتح الموقع في **سفاري** (لا كروم)
2. اضغط زر المشاركة ⎋ في الأسفل
3. اختر **إضافة إلى الشاشة الرئيسية**
4. أغلق سفاري، وافتح التطبيق من الأيقونة الجديدة
5. سجّل الدخول واضغط **السماح بالتنبيهات**

الأندرويد والكمبيوتر يعملان مباشرة من المتصفح بلا تثبيت.

من يفتح الموقع على آيفون بدون تثبيت ستظهر له شاشة تشرح هذه الخطوات، مع زر **المتابعة مؤقتًا بدون تنبيهات** حتى لا يُحبس خارج الموقع.

## استكشاف الأعطال

**لا يصل شيء إطلاقًا**
افتح **Actions** وانظر آخر تشغيل. إن كان أحمر ✗، افتحه واقرأ الخطأ. الأكثر شيوعًا `Missing VAPID_PUBLIC / VAPID_PRIVATE` — يعني أن أسماء الأسرار في الخطوة ٣ غير مطابقة.

**التشغيل أخضر لكن `0 notification(s) delivered`**
لا يوجد مشترك. تأكد أن `VAPID_PUBLIC_KEY` في `library.js` مملوء (الخطوة ٢)، وأن العضو ضغط "السماح" بعد رفع التعديل. تحقّق من وجود بيانات تحت `mdwnhLibrary/users/<الاسم>/push` في Firebase.

**كان يعمل ثم توقف لعضو واحد**
غالبًا حذف بيانات المتصفح أو أعاد تثبيت التطبيق. يسجّل خروجه ثم يدخل مجددًا ويسمح بالتنبيهات. الاشتراكات الميتة تُحذف تلقائيًا (أكواد 404/410).

**غيّرت مفاتيح VAPID**
كل الاشتراكات القديمة تصبح باطلة. على كل عضو تسجيل الخروج والدخول مجددًا. لا تغيّر المفاتيح بلا سبب.

**الجدولة توقفت بعد شهور**
GitHub يوقف الـ cron في المستودعات الخاملة تمامًا (٦٠ يومًا بلا أي نشاط). أي `git push` يعيد تفعيلها.

## ⚠️ ملاحظة أمنية مهمة

قاعدة بيانات Firebase مفتوحة حاليًا للقراءة والكتابة **بلا أي قيود**. أي شخص يعرف الرابط يستطيع قراءة أو تعديل أو حذف كل شيء فيها — بما في ذلك بيانات النقاط الموجودة من قبل.

هذا لم يتغيّر بسببي؛ كانت كذلك أصلًا. لكن الآن أصبحت تحوي مهام الفريق واشتراكات التنبيهات، فصارت أهم.

كذلك تسجيل الدخول بالبريد فقط **ليس تحققًا حقيقيًا** — من يعرف بريد نواف يستطيع فتح لوحة القائد. هذا مقبول لأداة داخلية بين زملاء، لكن لا تضع فيها شيئًا حسّاسًا.

إن أردت إغلاق هذه الثغرات لاحقًا، الحل هو Firebase Authentication مع قواعد أمان على قاعدة البيانات. أخبرني وقتها ونفعلها.
